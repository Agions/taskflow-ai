## 版本说明

### 版本号规则

本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范：

- **主版本号 (MAJOR)**: 不兼容的API修改
- **次版本号 (MINOR)**: 向下兼容的功能性新增
- **修订号 (PATCH)**: 向下兼容的问题修正

### 发布周期

- **主版本**: 每年1-2次，包含重大功能更新
- **次版本**: 每月1-2次，包含新功能和改进
- **修订版**: 根据需要发布，主要修复Bug

### 支持政策

- **当前版本**: 提供完整支持和更新
- **前一个主版本**: 提供安全更新和重要Bug修复
- **更早版本**: 不再提供支持，建议升级

## 迁移指南

### 从 1.1.x 升级到 1.2.x

#### 配置文件变更

```bash
# 备份现有配置
cp ~/.taskflow/config.json ~/.taskflow/config.json.backup

# 升级后重新验证配置
taskflow config validate

# 如有问题，可以重新初始化
taskflow config init
```

#### 命令行变更

```bash
# 旧命令 (1.1.x) - 项目创建功能（已移除）
taskflow init my-project --type web

# 新命令 (1.2.x) - 在现有项目中集成
cd existing-project
taskflow init
```

#### API变更

```typescript
// 旧API (1.1.x)
import { TaskManager } from 'taskflow-ai';
const manager = new TaskManager();

// 新API (1.2.x)
import { TaskManager } from 'taskflow-ai/core';
const manager = TaskManager.getInstance();
```

### 从 1.0.x 升级到 1.1.x

#### 重大变更

- 配置文件格式更新
- 命令行参数调整
- API接口重构

#### 升级步骤

1. 备份现有数据和配置
2. 卸载旧版本：`npm uninstall -g taskflow-ai`
3. 安装新版本：`npm install -g taskflow-ai@latest`
4. 重新初始化配置：`taskflow config init`
5. 迁移现有项目数据
