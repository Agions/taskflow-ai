### 详细修复内容

#### 1. string-utils.ts - snakeCase 函数
**问题**:
```typescript
// 修复前
.replace(/([A-Z])/g, '_$1')
.replace(/^-/, '')  // 只匹配第一个下划线
```

**修复**:
```typescript
// 修复后
.replace(/([A-Z])/g, '_$1')
.replace(/^_+/, '')  // 匹配开头的所有下划线
```

**测试用例**:
- `HelloWorld` → `hello_world` ✅
- `helloWorld` → `hello_world` ✅
- `XMLHttpRequest` → `x_m_l_http_request` ✅

#### 2. string-utils.test.ts - truncate 函数
**问题**: 测试期望值错误

**修复前**:
```typescript
expect(StringUtils.truncate('hello world', 5)).toBe('helli...'); // 错误
```

**修复后**:
```typescript
expect(StringUtils.truncate('hello world', 5)).toBe('he...'); // 正确
```

**逻辑**:
- `maxLength = 5`, `suffix = '...'`
- 保留字符 = `5 - 3 = 2`
- 结果 = `'he' + '...'` = `'he...'`

#### 3. built-in-tools.test.ts - 导入路径
**问题**: 相对路径错误

**修复前**:
```typescript
import { ToolDefinition } from '../../../types/tool';
import { ToolCategory } from '../../../types/tool';
```

**修复后**:
```typescript
import { ToolDefinition, ToolCategory } from '../../types/tool';
```

**路径分析**:
```
src/tools/__tests__/built-in-tools.test.ts
         │      └─ __tests__/
         └─ tools/

应该: ../../types/tool (正确)
实际: ../../../types/tool (多了一层 ../)
```
